# RAG 与知识库

> 🎯 目标：理解 RAG 的概念，学会搭建自己的 AI 知识库。

## 什么是 RAG？

**RAG（Retrieval-Augmented Generation，检索增强生成）** = 让 AI 在回答时参考你提供的资料，而不是只靠自己的训练数据。

```
普通 AI：凭记忆回答 → 可能不准确/过时
RAG AI：先查你的资料 → 再基于资料回答 → 更准确、更专业
```

## 为什么需要 RAG？

| 问题 | RAG 的解决方案 |
|------|--------------|
| AI 不知道你的公司内部信息 | 把公司文档喂给 AI |
| AI 知识有截止日期 | 实时检索最新信息 |
| AI 容易产生幻觉 | 基于真实资料回答，减少编造 |
| 通用 AI 不懂专业领域 | 注入领域专业知识库 |

## 小白能用的 RAG 工具

| 工具 | 特点 | 使用难度 |
|------|------|---------|
| Coze（扣子） | 可视化搭建，零代码 | ⭐ 简单 |
| Dify | 开源，功能丰富 | ⭐⭐ 中等 |
| FastGPT | 开源，专注知识库 | ⭐⭐ 中等 |
| NotebookLM | Google 出品，笔记+AI | ⭐ 简单 |

## 快速搭建个人知识库

```
1. 准备资料（PDF、Word、网页链接等）
2. 选择工具（推荐 Coze 或 NotebookLM）
3. 上传资料
4. 开始提问 — AI 会基于你的资料回答
```

## 用 Coze 搭建知识库 Bot 教程

```
第 1 步：准备你的资料
  → PDF、Word、TXT、网页链接

第 2 步：访问 https://www.coze.cn

第 3 步：创建知识库
  → 知识库 → 新建 → 上传文件

第 4 步：创建 Bot
  → 设定角色 Prompt：“你是 XX 领域专家，请基于知识库回答用户问题”
  → 添加插件：选择刚创建的知识库

第 5 步：测试并发布
  → 测试问答效果
  → 发布到网页/微信/飞书
```

## RAG 优化技巧

```
✅ 文档分块要合理（每块 300-500 字）
✅ 为每个文档添加标题和摘要
✅ 删除无关内容，保留核心信息
✅ 使用多种格式（文本 + 表格 + 列表）
✅ 定期更新知识库内容
✅ 设定 Bot 在不确定时明确说“知识库中没有相关信息”
```
